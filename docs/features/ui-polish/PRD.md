# mac fleet hub 重构 · 新功能 PRD

> 配合 `docs/features/ui-polish/` 高保真稿使用。本文件只写**新增功能**的需求与验收；接口为**建议形态**，最终 API 设计归 `/dev`（后端/agent）。
> 现有接口基线（`server/dashboard/app.js`）：`sessions` / `open{sessionId}` / `new{cwd}` / `projects` / `info` / `proxy` / `watch` / `reload` / `nodes.json`。

## F1 · 会话连接模型：选中 / 连接 / Bypass连接

**背景**：现状点击会话即直接打开终端。改为：点击只「选中」，由显式按钮触发连接，并区分普通连接与跳过权限连接。

**交互**
- 点击 / 点按会话行 = **仅选中**（高亮），不 attach、不打开终端。
- **选中后**展开两个按钮（桌面与移动一致，均为选中后显示，**不用 hover**）：
  - **连接**：打开 / 恢复该会话的 ttyd 终端，claude 以**正常权限**运行。
  - **Bypass连接**：同样打开 / 恢复，但 claude 以 `--dangerously-skip-permissions` 启动（跳过工具权限确认）。按钮用 danger（红）样式 + ⚠，终端头部显示「⚠ 跳过权限」徽标以示区分。

**前端改动**：会话行交互从「点击即连」改为「点击选中 + 按钮连接」；新增 Bypass连接 按钮与终端头部的 bypass 标识。

**后端改动（建议）**：`open` 与 `new` 增加一个布尔/枚举参数，例如
```
POST {id}/api/open  { sessionId, bypass: true }
POST {id}/api/new   { cwd, bypass: true }
```
`bypass:true` 时，后端启动 ttyd 所跑的 claude 命令带上 `--dangerously-skip-permissions`；缺省 `false` 保持现状。

**验收**
- 点击会话行不打开终端、仅高亮；再点按钮才连接。
- 「连接」开出的会话正常请求权限；「Bypass连接」开出的会话跳过权限确认。
- bypass 会话在终端头部有明显视觉区分。

**风险**：bypass 跳过安全确认，属高风险操作。建议 Bypass连接 首次点击二次确认（可选，待定）。

## F2 · 会话进程生命周期：返回保留 + 手动关闭

**背景**：现状从终端返回后，Mac 端 tmux/claude 进程本就保留（tmux 持久）。本需求把「关闭进程」变成用户**显式可控**的动作。

**交互**
- 从终端「返回」**不结束** Mac 端进程（保持现状的 tmux 持久行为）。
- **仅「连接过 / 当前有运行进程」的会话**显示终止入口（前端可由 `live` 或有无活动进程判断）；从未连接、无进程的会话不显示。
- 终止图标用**「终止 / 停止」语义**（如 ⏹），**不要用 ✕**——避免被误解为「删除会话」。选中会话后显示该图标。
- 点击 → **二次确认**（样式化弹窗 / 底部抽屉，非原生 confirm）。确认后结束该会话在 Mac 上的 claude / tmux 进程。
- **终止后会话条目仍保留在列表**（仅进程结束，非删除条目）；其在「活跃 / 全部」中的归类取决于 sessions 语义，待 `/dev` 定。

**后端改动（建议）**：新增结束会话接口，例如
```
POST {id}/api/close { sessionId }   // 或按现有终端 sid：{ sid }
```
后端 kill 对应 tmux 会话 / 终止该 claude 进程。

**验收**
- 返回终端不杀进程；再次进入仍是同一会话。
- 点 ✕ 并确认后，对应 Mac 进程被终止，列表更新。

## F3 · 登录有效期 30 天

**背景**：延长登录态，减少频繁重新认证。**属后端 / 部署，非 UI**。

**改动**：调整 Authelia 会话有效期（`server/authelia/configuration.yml` 的 `session` 配置——`expiration` / `inactivity` / `remember_me` 等），使登录态保持约 30 天。具体字段与安全权衡由 `/dev` / `/deploy` 决定。

**验收**：登录后 30 天内（在策略允许范围）无需重新认证。

---

## F4 · 退出登录

**背景**：当前无主动退出入口。

**交互**：用户区菜单（桌面：左下用户行 ▾ 菜单；移动：头部 ⋮ 菜单）新增「退出登录」。点击后清除会话并跳回登录页。

**后端改动（建议）**：前端指向 Authelia 的退出端点（如 portal 的 `/logout`，或反代上对应路由），登出后重定向到登录页。具体端点由 `/dev` 确认。

**验收**：点击退出后会话失效，再次访问需重新登录。
